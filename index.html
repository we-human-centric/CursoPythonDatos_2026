📋 Presentación del curso

🐍 Seminario de Análisis Exploratorio de Datos

Carrera de Software  ·  Abril – Mayo 2026

En 17 sesiones sincrónicas distribuidas en 4 semanas, los estudiantes construyen un Producto de Software Analítico completo: desde la formulación de la pregunta de negocio hasta un dashboard en Streamlit y una API documentada en FastAPI.

1. Resultados de aprendizaje

1.1 Resultados de aprendizaje globales

  • Aplica los fundamentos de Python y librerías especializadas para diseñar, estructurar y ejecutar procesos de análisis de datos que respondan a problemas de negocio reales.
  • Integra datos provenientes de distintas fuentes (archivos, bases de datos SQL, APIs) y construye pipelines reproducibles que aseguren consistencia, calidad y confiabilidad.
  • Analiza y visualiza datos mediante técnicas estadísticas y exploratorias, generando hallazgos útiles para la toma de decisiones.
  • Implementa modelos predictivos básicos con Python y evalúa su desempeño con métricas apropiadas.
  • Entrega resultados en forma de productos de software analítico (dashboards y/o APIs) alineados a estándares de la industria.
  • Demuestra pensamiento crítico y capacidad de comunicación técnica, presentando hallazgos, limitaciones y propuestas de mejora de manera fundamentada.

2. Estructura modular

El seminario se divide en 4 módulos que se despliegan de forma incremental en 17 sesiones sincrónicas. Cada módulo avanza un paso más en el ciclo completo de un proyecto analítico.

MóduloNombreContenidos principalesSesiones
M1Introducción a Python y kick-off del proyectoFundamentos de Python, sintaxis, estructuras de control, funciones, estructura de proyecto y pregunta de negocio.Días 1 – 4
M2Manipulación y exploración de datosProgramación orientada a objetos, Pandas (lectura, exploración, limpieza) y primeros pasos de análisis.Días 5 – 7
M3Modelado supervisado: del EDA al primer modeloVisualizaciones, regresión lineal/polinomial + VIF, regresión logística + SMOTE, árboles de decisión y métricas de evaluación.Días 8 – 12
M4Ciencia de datos aplicada y producto analíticoEnsembles (Random Forest), pruebas de hipótesis, dashboards con Streamlit, APIs con FastAPI, buenas prácticas y presentación final.Días 13 – 17

8. Recursos y herramientas recomendadas

8.1 Entorno y lenguajes

  • Python 3.11+, pip / venv / conda.
  • VS Code con extensiones Python, Jupyter, GitLens y Black.
  • Git 2.40+ y cuenta en GitHub.

8.2 Librerías principales

  • Análisis de datos: NumPy, Pandas, SciPy, statsmodels.
  • Visualización: Matplotlib, Seaborn, Plotly.
  • Machine learning: scikit-learn, LightGBM (opcional).
  • Serialización y validación: joblib, pydantic, pandera.
  • Productos analíticos: Streamlit, FastAPI, Uvicorn.
  • Testing y calidad: pytest, black, ruff.
