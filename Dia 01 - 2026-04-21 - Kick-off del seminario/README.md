## Introducción

Bienvenidos al Seminario de Análisis Exploratorio de Datos. En esta primera sesión vamos a alinearnos en la metodología del curso, conocer los casos disponibles, conformar los equipos de trabajo y dejar el entorno de desarrollo listo para programar desde el próximo día. No se requiere experiencia previa con Python más allá de lo que viste durante la carrera.

## Objetivos de aprendizaje

Al finalizar esta sesión serás capaz de:

- Comprender el alcance, metodología y evaluación del seminario.
- Identificar el caso de estudio que trabajarás durante las 4 semanas.
- Conformar un equipo de 3 personas con roles definidos.
- Dejar tu entorno de desarrollo listo (Python, VS Code/Jupyter, Git y GitHub).

## Contenidos de la sesión

### 1. Presentación y contexto

Docente, estudiantes, reglas de la sesión virtual y vinculación con el proceso complexivo de titulación. Importancia del seminario y homologación con la Maestría en Big Data y Ciencia de Datos.

### 2. Metodología y evaluación

40 horas sincrónicas en 17 sesiones a lo largo de 4 semanas (la mayoría de 2h30, la semana 3 de 2h). Trabajo asincrónico: ~20 horas por semana. Entregables semanales (S1 a S4) y defensa oral del Producto de Software Analítico ante un tribunal.

### 3. Recorrido por los 15 casos de la matriz

Revisión rápida de cada caso disponible (retail, música, turismo, banca, salud pública, deportes, recomendación, movilidad, sismos, vuelos, StackOverflow, Airbnb, reseñas, vehículos usados).

### 4. Conformación de equipos

Equipos de 3 estudiantes. Roles sugeridos: Data Lead (pipeline y calidad), Model Lead (EDA y modelado) y Product Lead (dashboard/API y documentación). Los roles rotan si el grupo lo prefiere.

### 5. Instalación del entorno

Python 3.11+, VS Code con las extensiones Python y Jupyter, Git, cuenta en GitHub, entorno virtual con `python -m venv .venv` y primer commit.

## Actividad práctica

1) Instala Python y verifica con `python --version`.

2) Abre VS Code, crea una carpeta llamada `seminario-eda/` y dentro un archivo `hola.py` con:

```python
print("Hola, seminario EDA")
```

3) Ejecútalo desde la terminal: `python hola.py`.

4) Crea un notebook `hola.ipynb`, escribe la misma línea en una celda y ejecuta.

5) Inicializa Git: `git init`, `git add .`, `git commit -m "primer commit"`.

## Trabajo en grupo sobre el caso asignado

- Conformen el equipo de 3 personas y elijan el caso de la matriz que van a trabajar.
- Asignen roles (Data Lead, Model Lead, Product Lead).
- Creen un repositorio en GitHub compartido entre los 3 (pueden hacerlo privado).
- Agreguen un `README.md` inicial con: nombre del grupo, integrantes, caso elegido y enlace al dataset.

## Entregable del día

Captura del entorno instalado (salida de `python --version` y `git --version`) y enlace al repositorio de GitHub con el `README.md` inicial.
