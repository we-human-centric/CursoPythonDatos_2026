1. ¿Por qué evaluar?

Sin métricas, el modelo es pura especulación. Un modelo puede parecer «funcionar» en los datos que lo entrenaron, pero fallar completamente en datos nuevos. Por eso separamos train y test. Por eso usamos validación cruzada. Por eso reportamos múltiples métricas — ninguna sola cuenta la historia completa.

Esta semana vimos tres enfoques: regresión lineal (números), regresión logística (probabilidades binarias) y árboles de decisión (reglas no-lineales). Hoy aprendemos a juzgarlos justa y correctamente.

2. Métricas de regresión

Cuando la variable objetivo es continua:

2.1 Mean Absolute Error (MAE)

MAE = (1/n) Σ |yi − ŷi|

En unidades del target. Robusto a outliers (no penaliza cuadrados). Si MAE = 5 y el target es en miles de USD, te equivocas en promedio 5,000 USD por predicción.

2.2 Mean Squared Error (MSE) y Root Mean Squared Error (RMSE)

MSE = (1/n) Σ (yi − ŷi)2
RMSE = √MSE

MSE penaliza más fuerte los errores grandes (cuadrados). RMSE está en unidades del target, lo que lo hace comparable.

2.3 R² (Coeficiente de determinación)

Ya lo vimos. Proporción de varianza explicada. Rango [0, 1]. Fácil de interpretar pero inflado por variables adicionales.

2.4 MAPE (Mean Absolute Percentage Error)

MAPE = (1/n) Σ |yi − ŷi| / |yi|

Error porcentual. Peligro: si yi ≈ 0, el denominador explota. Útil cuando quieres entender error relativo.

3. Matriz de confusión

Para clasificación binaria, la matriz de confusión resume todos los resultados en una tabla 2x2:

Predicción: Positivo Predicción: Negativo
Real: Positivo TP (Verdadero Positivo) FN (Falso Negativo) — error tipo II
Real: Negativo FP (Falso Positivo) — error tipo I TN (Verdadero Negativo)

Todo lo demás se construye a partir de estos 4 valores.

4. Accuracy, Precision, Recall, F1, Specificity

4.1 Accuracy

Accuracy = (TP + TN) / (TP + FP + FN + TN)

Proporción total de predicciones correctas. Engañosa si hay desbalance de clases. Evítalo cuando un clase es rara.

4.2 Precision

Precision = TP / (TP + FP)

De todas las predicciones positivas, ¿cuántas fueron correctas? Si es detector de spam, precision alta = pocos correos buenos marcados como spam.

4.3 Recall (Sensibilidad)

Recall = TP / (TP + FN)

De todos los positivos reales, ¿cuántos detecté? Si es detección de cáncer, recall alto = pocos casos reales sin detectar.

4.4 F1 (Media armónica de Precision y Recall)

F1 = 2 · (Precision · Recall) / (Precision + Recall)

Balance entre los dos. Si precision es 0.9 pero recall es 0.1, F1 = 0.18 (malo). Útil cuando ambos importan.

4.5 Specificity

Specificity = TN / (TN + FP)

De todos los negativos reales, ¿cuántos identificaste correctamente? Complemento de recall.

4.6 Tabla decisional

  • Spam: falsos positivos caros (pierdes usuarios satisfechos) → prioriza Precision.
  • Cáncer: falsos negativos caros (muertes evitables) → prioriza Recall.
  • Balance general: F1.

5. Curva ROC y AUC

La curva ROC (Receiver Operating Characteristic) es un gráfico que muestra la relación entre tasa de verdaderos positivos (Recall) y tasa de falsos positivos (1 − Specificity) a medida que varías el umbral de clasificación.

El área bajo la curva (AUC) resume todo en un número: 0.5 (modelo aleatorio) a 1.0 (perfecto). AUC mide la probabilidad de que el modelo clasifique correctamente un par aleatorio (positivo real vs negativo real).

Ventaja: AUC es robusto al desbalance moderado — no es engañado por una clase rara.

6. Curva Precision-Recall (para desbalance fuerte)

Cuando hay desbalance extremo (ej. fraude: 0.1%), la curva ROC puede parecer excelente aunque el modelo sea malo. La curva Precision-Recall es más honesta en estos casos: dibuja Precision vs Recall variando el umbral.

AUC-PR es el área bajo esa curva. Cuando hay desbalance, AUC-PR dice la verdad mejor que AUC-ROC.

7. Threshold tuning

El umbral default es 0.5 (la clase más probable gana), pero es ajustable. Si los falsos negativos son 10 veces más caros, baja el umbral a 0.3: serás más agresivo en predecir la clase positiva. 

from sklearn.metrics import precision_recall_curve

probs = modelo.predict_proba(X_test)[:, 1]
precision, recall, thresholds = precision_recall_curve(y_test, probs)

# Buscar threshold que maximice F1
f1_scores = 2 * (precision * recall) / (precision + recall + 1e-10)
best_idx = np.argmax(f1_scores)
best_threshold = thresholds[best_idx] if best_idx < len(thresholds) else 0.5

print(f"Mejor umbral: {best_threshold:.3f}")

8. Validación cruzada (k-fold)

Divides el dataset en k partes, entrenas k modelos (cada uno deja una parte afuera como test), y promedias las métricas. Elimina la variabilidad por suerte en un único train/test split. 

from sklearn.model_selection import cross_val_score, StratifiedKFold

skf = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
scores = cross_val_score(modelo, X, y, cv=skf, scoring='accuracy')

print(f"Scores: {scores}")
print(f"Mean: {scores.mean():.3f} ± {scores.std():.3f}")

StratifiedKFold mantiene la proporción de clases en cada fold (importante con desbalance).

10. Bias-Variance Tradeoff

Un modelo con alto bias (subajuste) es demasiado simple: subestima la complejidad. Un modelo con alta variance (sobreajuste) es demasiado flexible: memoriza ruido.

La validación cruzada te muestra el tradeoff: si train score sube pero CV score baja, has cruzado la línea del sobreajuste.
