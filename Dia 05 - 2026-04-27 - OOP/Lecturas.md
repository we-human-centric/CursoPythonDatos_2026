# Lecturas adicionales · Día 05 · Programación Orientada a Objetos (OOP)

Curaduría de material en español para profundizar después de la clase.

## Video de la clase

- **[Programación Orientada a Objetos en Python — mouredev](https://www.youtube.com/watch?v=Tje_s_j7Suk)**
  *Por qué verlo:* Cubre los cuatro pilares de OOP (encapsulamiento, herencia, polimorfismo, abstracción) con ejemplos claros en español. Recomendado ver los primeros 20 minutos antes de la clase.

## Artículos técnicos

1. **[Programación Orientada a Objetos en Python 3](https://realpython.com/python3-object-oriented-programming/)** — Real Python
   *Por qué leerlo:* Tutorial completo en inglés con muchos ejemplos. Cubre todo lo de la clase y más. El más completo disponible.

2. **[Clases y Objetos en Python](https://docs.python.org/es/3/tutorial/classes.html)** — Documentación oficial de Python (en español)
   *Por qué leerlo:* La fuente primaria. Explica `__init__`, herencia y el modelo de datos de Python.

3. **[Métodos mágicos en Python](https://rszalski.github.io/magicmethods/)** — Rafe Kettler
   *Por qué leerlo:* Referencia completa de todos los métodos dunder. Útil como consulta cuando necesitas implementar `__add__`, `__iter__`, etc.

## Artículos de negocios y aplicación

1. **[Por qué OOP sigue siendo relevante en Data Science](https://towardsdatascience.com/object-oriented-programming-for-data-scientists-build-your-ml-estimator-7da416751f64)** — Towards Data Science
   *Por qué leerlo:* Cómo scikit-learn usa OOP para construir estimadores. Verás el patrón `fit/predict` explicado desde la perspectiva de clases.

2. **[Design Patterns en Python para Data Engineers](https://www.databricks.com/blog/2023/04/05/hacking-databricks-lakehouse-patterns-python.html)** — Databricks
   *Por qué leerlo:* Casos reales donde OOP organiza pipelines de datos complejos en producción.

## Para explorar más

- **`dataclasses`**: forma moderna de definir clases en Python con menos código: `from dataclasses import dataclass`
- **`abc`**: clases abstractas para forzar implementación de métodos en subclases
- **`__slots__`**: optimización de memoria para clases con muchos atributos
