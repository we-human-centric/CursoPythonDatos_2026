Introducción

Cerramos la primera semana montando la estructura profesional del proyecto. Un proyecto de datos bien planteado ahorra semanas de trabajo; uno mal planteado se descarrila en la fase de modelado. Hoy es el día para definirlo bien.

Contenidos de la sesión

1. CRISP-DM: entendimiento de negocio a despliegue

Flujo: Entendimiento del negocio → Datos → Preparación → Modelado → Evaluación → Despliegue. Veremos cómo se mapea a los 4 módulos del curso.

2. La pregunta de negocio

Tres familias: medir (KPIs, reportes), explicar (qué factores influyen), predecir (estimar valores o clasificar). Debe ser específica, respondible con los datos disponibles, y relevante.

3. Estructura de carpetas profesional

seminario-eda/
├─ data/raw/       # originales, nunca modificar
├─ data/processed/ # limpios
├─ notebooks/      # numerados 01_, 02_, ...
├─ src/            # código reutilizable
├─ reports/        # figuras y documentos
├─ README.md
└─ requirements.txt

4. README profesional

Responde en 1 minuto: ¿qué hace?, ¿cómo se instala?, ¿cómo se ejecuta?, ¿quién lo mantiene? Usa badges, capturas, ejemplos.

5. Entorno virtual y dependencias

python -m venv .venv && source .venv/bin/activate && pip install -r requirements.txt

6. Ficha del dataset

Documento con: origen, licencia, tamaño, tipos por columna, % nulos, limitaciones conocidas.

Actividad práctica

  1. Creen entorno virtual y estructura de carpetas.
  2. Escriban pregunta de negocio con justificación (1 párrafo).
  3. Carguen dataset, ejecuten df.head(), df.info(), df.shape, df.describe().
  4. Escriban ficha en docs/ficha_dataset.md.
